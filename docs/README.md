# Documentation

AI Health Insurance Navigator for Newcomers to Canada

Choosing the right health insurance plan can be difficult for anyone, but it is especially challenging for newcomers to Canada who may not be familiar with how the country’s healthcare and private insurance systems work. Many immigrants, international students, and temporary workers arrive in Canada without a clear understanding of what provincial health coverage includes, what private insurance plans are available, and which plans actually meet their needs. The result is that people often purchase insurance that is either unnecessarily expensive or insufficient for their medical circumstances. This project was built to address that problem by creating a web application that guides users through the process of selecting an appropriate health insurance plan using AI-assisted analysis.

The goal of this project is to create a full-stack web platform that helps first-time insurance buyers understand their options and receive personalized recommendations based on their individual situation. Instead of forcing users to manually compare dozens of complicated insurance policies, the system analyzes their demographic and medical information and then evaluates which insurance plans are the best fit. The application ultimately returns the three most suitable plans along with a clear explanation of why those plans were recommended. To make the recommendation easier to understand, the platform also generates a simple visualization that illustrates the long-term financial impact of purchasing insurance versus not having coverage.

At a high level, the application works by combining user-submitted data, a repository of insurance plan information, and an AI-driven reasoning pipeline that compares the two. When a user first visits the website, they are prompted to provide basic demographic information such as their name, age, and nationality. This information helps the system understand eligibility requirements and general risk categories that may apply to the user. After submitting demographic information, users are asked to provide details about their medical history. They can either write a description of their health background or upload documents such as medical records, physician notes, or previous health reports. This information is important because certain insurance plans cover specific conditions or offer better protection for certain health risks.

All of this user input is stored in a database that preserves the original information exactly as it was submitted. The system does not modify or overwrite this data, since the raw input may be useful for auditing or future processing. Instead, the application uses a separate AI processing stage to analyze the information. At this stage, an LLM API agent reviews the user’s demographic data and medical history and converts it into structured information that can be used for analysis. For example, the system may identify relevant medical conditions, risk factors, medications, or lifestyle indicators such as smoking status. The output of this step is a structured profile that summarizes the user’s health background in a consistent format.

While user information is being processed, the application also maintains a structured database of available insurance plans. These plans originate from a repository of markdown files that contain detailed descriptions of each insurance policy. Each file typically includes information such as the plan name, provider, monthly premium, deductible, coverage categories, eligibility rules, and other relevant details. Because these files are written for humans rather than machines, the application uses an AI extraction process to convert the markdown content into structured data. The LLM reads each document, identifies the relevant plan attributes, and produces a standardized representation of the plan. This ensures that every plan in the system can be compared consistently.

Once both the user profile and the insurance plan data have been structured, the system evaluates which plans best match the user’s needs. The AI reasoning process considers several factors when making recommendations. These include the user’s eligibility for the plan, the degree to which the plan covers relevant medical conditions, the cost of the plan relative to its coverage, and the level of risk protection the plan provides. By comparing the user’s health profile with the coverage features of each available insurance plan, the system determines which options are most appropriate. The final result of this evaluation is a ranked list of the top three insurance plans that best match the user’s situation.

However, simply listing recommended plans is not always enough to help users make a confident decision. For that reason, the system includes a final stage where the AI generates clear, user-friendly explanations for each recommendation. The AI reviews the details of the selected plans and produces short summaries that highlight the key characteristics of each option. These summaries explain what the plan covers, what the monthly costs look like, and why the plan may be a good fit for the user’s health profile. The goal is to translate complex insurance terminology into simple language that users can understand.

In addition to plan summaries, the system generates a visualization that illustrates the potential financial consequences of insurance decisions. Specifically, the application creates a projection showing the expected cost of healthcare over a ten-year period under two different scenarios. In the first scenario, the user purchases the highest-ranked insurance plan recommended by the system. In the second scenario, the user does not purchase insurance and instead pays healthcare expenses out of pocket. The visualization displays the cumulative cost for each scenario across multiple years, helping users see how insurance can provide financial protection over time. This comparison makes it easier for people who are unfamiliar with insurance systems to understand the long-term value of coverage.

Another important design goal of the system is scalability. The architecture is designed so that it can support a growing number of users and insurance plans without requiring significant changes to the underlying infrastructure. Because insurance plan data is extracted automatically from markdown files, new plans can be added to the system simply by adding additional documents to the repository. The AI processing pipeline will detect and process these new plans automatically, ensuring that the database stays up to date. Similarly, user data processing and recommendation generation are handled by modular backend services, which means the system can handle increasing workloads as more users interact with the platform.

To ensure flexibility, the backend uses an abstraction layer for AI services. This means the application can work with multiple LLM providers without changing the overall system architecture. If one provider becomes unavailable or a different model becomes more cost-effective, the system can switch providers without rewriting the application logic. This design choice helps make the platform more reliable and future-proof as AI technology evolves.

Ultimately, this project demonstrates how AI can simplify complicated financial decisions by transforming large amounts of unstructured information into clear, personalized recommendations. By combining user data analysis, automated document processing, and explainable AI outputs, the platform provides a practical tool for newcomers who may feel overwhelmed by Canada’s insurance landscape. The long-term vision is to expand this system with additional features such as multilingual support, province-specific insurance recommendations, and real-time updates from insurance providers. With these improvements, the platform could serve as a comprehensive guide for anyone navigating the Canadian health insurance system for the first time.
